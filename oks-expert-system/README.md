# ОКС Эксперт - Система управления объектами капитального строительства и эксплуатации

## Описание

Единая веб-платформа для специалистов отдела капитального строительства (ОКС) и службы эксплуатации. Система объединяет финансовый учёт, управление задачами и визуальный контроль состояния объектов на карте Москвы и Московской области.

### Целевая аудитория
- Начальники участков
- Инженеры ОКС
- Специалисты по эксплуатации
- Руководители проектов
- Подрядчики
- Управляющие компании

## Технологический стек

### Backend
- **Python 3.11** + **FastAPI** - современный асинхронный фреймворк
- **PostgreSQL 15** + **PostGIS** - геоданные в формате WGS84
- **Redis 7** - кэширование и очереди уведомлений
- **SQLAlchemy 2.0** - ORM с поддержкой геоданных
- **Pydantic 2** - валидация данных
- **JWT** - аутентификация

### Frontend
- **Vue 3** - реактивный фреймворк
- **Pinia** - управление состоянием
- **Vue Router** - навигация
- **Vite** - сборка
- **PWA** - прогрессивное веб-приложение
- **Yandex Maps API v3** - картография

### Infrastructure
- **Docker** + **Docker Compose** - контейнеризация
- **Nginx** - раздача статики

## Быстрый запуск

### Требования
- Docker 20+
- Docker Compose 2.0+
- 4GB RAM минимум
- 10GB свободного места

### Установка и запуск

```bash
# Перейдите в директорию проекта
cd oks-expert-system

# Запустите все сервисы одной командой
docker compose up -d

# Проверьте статус контейнеров
docker compose ps

# Просмотрите логи
docker compose logs -f
```

### Открытие в браузере

После запуска откройте:
- **Фронтенд**: http://localhost:3000
- **Бэкенд API**: http://localhost:8000
- **API документация**: http://localhost:8000/docs

### Остановка

```bash
docker compose down
```

### Сброс данных

```bash

```

## API Endpoints

### Объекты
- `GET /api/objects` - Список объектов с фильтрацией
- `GET /api/objects/{id}` - Детали объекта
- `POST /api/objects` - Создание объекта
- `PUT /api/objects/{id}` - Обновление объекта
- `DELETE /api/objects/{id}` - Удаление объекта
- `GET /api/objects/{id}/documents` - Документы объекта
- `GET /api/objects/{id}/defects` - Дефекты объекта
- `GET /api/objects/{id}/tasks` - Задачи объекта
- `GET /api/objects/nearby` - Объекты рядом с координатами

### Дефекты
- `GET /api/defects` - Список дефектов
- `GET /api/defects/{id}` - Детали дефекта
- `POST /api/defects` - Создание дефекта
- `PUT /api/defects/{id}` - Обновление дефекта
- `DELETE /api/defects/{id}` - Удаление дефекта

### Дашборд
- `GET /api/dashboard/stats` - Статистика дашборда
- `GET /api/dashboard/map-objects` - Объекты для карты
- `GET /api/dashboard/recent-defects` - Последние дефекты
- `GET /api/dashboard/overdue-tasks` - Просроченные задачи

### Аутентификация
- `POST /api/auth/login` - Вход в систему

### Пользователи
- `GET /api/users` - Список пользователей

## Безопасность

- JWT токены для аутентификации
- CORS настройка для ограничения доменов
- Валидация всех входных данных через Pydantic
- Защита от SQL-инъекций через параметризированные запросы
- XSS защита через экранирование вывода
- HTTPS рекомендуется для production

## Настройка для production

1. Измените пароли в `docker-compose.yml`:
   ```yaml
   POSTGRES_PASSWORD: your-secure-password
   JWT_SECRET: your-super-secret-key
   ```

2. Получите API ключ Яндекс.Карт и укажите в `frontend/index.html`:
   ```html
   <script src="https://api-maps.yandex.ru/v3/?apikey=YOUR_API_KEY&lang=ru_RU"></script>
   ```

3. Настройте S3 хранилище для файлов

4. Включите HTTPS через reverse proxy (nginx/traefik)

```bash
pip install -r requirements.txt

```

```bash
cd frontend
npm install
npm run dev
```
